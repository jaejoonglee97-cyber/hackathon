# UI 가이드(미니) — 열매똑똑 해커톤 보드 (MVP)

## 1) 디자인 목표
- ‘성과 과시’보다 ‘학습/검증/협력’을 더 크게 보이게 한다.
- 실무자가 한 번에 훑도록 **카드 기반 + 명확한 필터/정렬**을 기본으로 한다.

## 2) 컴포넌트 정의(최소 세트)
### Team Card
- 필수: 팀명, 스테이지 배지, 최근 업데이트(1줄), Help/Insight 카운트, 기여 배지
- 금지: 점수/석차 노출(기본)

### Help Card
- 상태(Open/In progress/Resolved) + 태그 + “원하는 도움 1줄”
- CTA: ‘피드백 남기기’, ‘해결 제안’

### Insight Card
- “틀렸던 가정 1개”, “다음에 검증할 것 1개” 고정

### Rubric Table(심사)
- 항목/설명/점수 입력
- 채점은 자동 저장 + 제출 버튼으로 확정

## 3) 상태/배지 표준
- 스테이지: 도입 / 검증 / 완성 (운영자가 명칭 변경 가능)
- 기여 배지: 피드백 n회, Help 해결 n회, 템플릿 공유 n회

## 4) 텍스트 규칙(쉽게)
- 한글 기준: 문장 짧게, 전문용어 최소화
- “베끼기”라는 표현 대신 “재사용(출처+변경점)” 사용
- 개인정보 안내는 모든 입력 폼 상단에 1줄 고정

## 5) 접근성(필수)
- 포커스 표시(키보드 사용자) — Focus Visible 권고  
  https://www.w3.org/WAI/WCAG22/Understanding/focus-visible.html
- 대비/글자 크기/클릭 영역 고려(WCAG 2.2 권고)  
  https://www.w3.org/TR/WCAG22/
