# UX 스펙(브리프) — 열매똑똑 해커톤 보드 (MVP)

## 1) 사용자 여정(핵심)
1. 참가자: 팀 생성 → Why/가설 작성 → Help 올림 → 피드백 반영 → 검증 로그 → 제출
2. 멘토: Help 탐색 → 피드백 → 해결 제안 → “도움 됨” 지표 확인
3. 심사위원: 제출 버전 열람 → 루브릭 채점(비공개) → 코멘트(공개/비공개)
4. 운영자: 스테이지/마감 설정 → 신고 처리 → 심사 배정 → 결과 집계 → 공개 정책 관리

## 2) 필수 플로우(화면 전환)
- 메인(/) → 팀(/teams/[id]) → Help 작성 → 알림 수신
- 제출(/submissions) → 제출 스냅샷 확정 → 심사(/judge)로 배정
- Public Viewer: 메인(공개 범위) → 팀 페이지(공개 섹션만)

## 3) IA(정보 구조)
- / : 대시보드(필터/정렬/공지/공동목표)
- /teams/[id] : 팀 페이지(Why→가설→해결→데모→검증→확산)
- /help : Help 카드 모아보기(태그/상태/최근순)
- /submissions : 제출 버전 목록(운영/심사)
- /judge : 심사위원 대시보드
- /admin : 운영 콘솔(스테이지/마감/루브릭/공개/신고)

## 4) 폼 설계(실무자 친화)
- 텍스트 입력은 “예시 문장(placeholder)”을 반드시 제공
- 긴 글 강요 금지: 1문장/3개/5줄 같은 제한으로 부담 낮추기
- 저장은 자동저장(초안) + 제출 스냅샷(고정) 분리

## 5) 접근성 체크(최소)
- 키보드 포커스 표시(포커스 링)
- 탭 순서 논리적(상단→필터→콘텐츠)
- 버튼/링크 구분 명확(색만으로 구분하지 않기)
